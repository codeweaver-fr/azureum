# Sprint 005 — Structure fonctionnelle du produit

## Statut

🟡 Validation prononcée — verrouillage Git en attente

## 1. Objectif

Le Sprint 005 définit comment AZUREUM organise ses espaces, ses informations et ses parcours afin de rendre utilisables les capacités de la V1.

Il ne redéfinit ni le périmètre fonctionnel du Sprint 003-r1 ni les règles métier du Sprint 004-r1.

## 2. Périmètre

Le Sprint couvre uniquement :

- les espaces du produit ;
- les pages et vues nécessaires ;
- l'architecture de l'information ;
- la navigation entre les espaces ;
- les parcours principaux ;
- les informations et actions visibles selon les acteurs ;
- les états perceptibles nécessaires à la compréhension de l'expérience.

## 3. Livrables

- `docs/specifications/product-structure-v1.md`
- `docs/sprints/sprint-005-product-structure.md`

Le premier document constitue le livrable principal du Sprint.

## 4. Hors périmètre

Le Sprint ne définit pas :

- de nouvelle capacité fonctionnelle ;
- de nouvelle règle métier ;
- le modèle de données ;
- les technologies, composants ou API ;
- le code ;
- les routes techniques ;
- les maquettes graphiques détaillées.

## 5. Règle de cohérence

Aucune décision du Sprint 005 ne peut modifier une capacité du Sprint 003-r1 ni une règle métier du Sprint 004-r1.

Toute contradiction doit être signalée au Product Owner et ne doit jamais être résolue implicitement.

## 6. Critères de clôture

Le Sprint est clôturable lorsque :

- les espaces fonctionnels sont définis ;
- les pages et vues nécessaires sont inventoriées ;
- l'architecture de l'information est cohérente avec la vision ;
- les parcours principaux couvrent les cas d'usage indispensables ;
- les informations, actions et états perceptibles sont définis au bon niveau ;
- la traçabilité avec les Sprints 003-r1 et 004-r1 est vérifiée ;
- aucun choix métier ou technique hors périmètre n'est introduit ;
- le Product Owner valide le livrable ;
- le verrouillage documentaire et Git est réalisé conformément à la gouvernance.
